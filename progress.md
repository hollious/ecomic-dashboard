## [서버 관리 가이드]

### 두 서버 모두 끄고 다시 켜는 방법

#### 1. 서버 중지
- 터미널에서 `Ctrl + C` 누르기 (각 터미널마다)

#### 2. 서버 시작
**방법 1: 두 개의 터미널 사용**
- 터미널 1: `cd ~/Downloads/Claude_Code_Test/Test && npm run proxy`
- 터미널 2: `cd ~/Downloads/Claude_Code_Test/Test && python3 -m http.server 8000`

**방법 2: 한 번에 실행**
- `cd ~/Downloads/Claude_Code_Test/Test && npm run dev`

#### 3. 브라우저에서 확인
- `http://localhost:8000` 열기

#### 4. 코드 수정 후 서버 재시작
- 코드를 수정한 후에는 반드시 프록시 서버를 재시작해야 함
- `Ctrl + C`로 중지 후 `npm run proxy`로 다시 시작

#### 5. 브라우저 캐시 비우기
- `Ctrl + Shift + R` (Mac: `Cmd + Shift + R`)로 강제 새로고침
- 또는 개발자 도구(F12) → Application → Local Storage에서 캐시 삭제

#### 6. API 리밋 주의사항
- Alpha Vantage 무료 API는 분당 5회 요청 제한
- 새로고침 시 캐시를 삭제하지 않으므로 API 리밋 방지
- 처음 로드 시에는 여러 지표를 순차적으로 요청하므로 시간이 소요됨

### [해결된 이슈들]
- ~~이슈: `script.js`가 업데이트되지 않아 대시보드가 작동하지 않음~~
  - 해결책: `script.js`를 완전히 재작성하여 모든 기능을 연결함
- ~~이슈: FRED API 호출 시 404 에러 발생~~
  - 원인: 프록시 서버에서 URL 경로에 `/fred` 부분이 빠져 있음
  - 해결책: `proxy-server.js`에서 FRED API URL에 `/fred` 경로 추가
- ~~이슈: FRED API 호출 시 400 Bad Request 에러 발생~~
  - 원인: 프록시 서버에서 `req.path` 사용으로 쿼리 파라미터 누락
  - 해결책: `proxy-server.js`에서 모든 API 프록시의 `req.path`를 `req.url`로 변경
- ~~이슈: Canvas 중복 사용 에러~~
  - 원인: 차트 업데이트 함수에서 반환값을 저장하지 않음
  - 해결책: `script.js`에서 모든 차트 업데이트 함수가 반환값을 저장하도록 수정
- ~~이슈: VIX, 나스닥, S&P 500 지표가 NaN/N/A로 표시됨~~
  - 원인: Alpha Vantage API 응답 형식 처리 불완전
  - 해결책: `script.js`에서 다양한 Alpha Vantage 응답 형식을 처리하도록 수정
- ~~이슈: Chart.js Canvas 재사용 오류 (새로고침 시)~~
  - 원인: `createKRWIndexChart`와 `createSparkline` 함수에서 차트 인스턴스를 반환하지 않음
  - 해결책: `utils/chart.js`에서 모든 차트 생성 함수가 차트 인스턴스를 반환하도록 수정
- ~~이슈: Alpha Vantage API 빈 데이터 및 제한 메시지 처리~~
  - 원인: 빈 데이터나 API 제한 메시지를 적절히 처리하지 않음
  - 해결책: `script.js`에서 데이터 형식 검증 강화 및 에러 메시지 표시 추가
- ~~이슈: Alpha Vantage API 리밋 초과~~
  - 원인: 여러 지표를 동시에 요청하여 분당 5회 제한 초과
  - 해결책: 각 지표 요청 사이에 1초 딜레이 추가 및 캐시 전략 개선
- ~~이슈: 환율 추이를 볼 수 없음~~
  - 원인: Exchange Rate API 무료 버전은 최신 환율만 제공
  - 해결책: FRED API에서 환율 시계열 데이터 가져오기 및 기간별 버튼 추가
- ~~이슈: Alpha Vantage API 빈 데이터 문제~~
  - 원인: Alpha Vantage 무료 API가 지수 데이터를 제공하지 않음
  - 해결책: Alpha Vantage 지표들 일시 비활성화
- ~~이슈: 환율 차트에 점 하나만 표시됨~~

  - 해결책: 차트 데이터 처리 로직 수정
- ~~이슈: 원화 인덱스가 표시되지 않음~~
